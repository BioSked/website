# biosked.com — multilingual marketing site (production)

Live site for **biosked.com**: English (default, unprefixed) plus `fr`, `fr-ch`,
`de`, `nl`, `it`. Astro 5 + React 19 + Tailwind 4, static build, deployed to
GitHub Pages on every push to `main` (`.github/workflows/deploy.yml`).

**Rollback to the English-only site: see [ROLLBACK.md](ROLLBACK.md).**

## Locale map

| Locale | Path | State |
|---|---|---|
| English | `/` | Full site (reference design) |
| Français | `/fr/…` | Full market site: landing pages, cas clients, 46 posts, ressources |
| Français (Suisse) | `/fr-ch/…` | Conversion path (home, pricing CHF, demo, devis, sécurité, mentions); deep content shared with `/fr` |
| Deutsch / Nederlands / Italiano | `/de` `/nl` `/it` | Home, demo, pricing, getquote, references, security localized; deeper pages fall back to English |

## Key architecture

- `src/i18n/locales.ts` — locale metadata, cross-locale path mapping (`altPathFor`),
  hreflang + canonical policy. `navLinks.ts` / `footerLinks.ts` / `homeContent.ts` /
  `market/*.ts` carry per-locale nav, footer and page copy.
- Language auto-detection: `src/components/LocaleSuggest.astro` (browser language,
  homepage-only silent switch, suggestion banner elsewhere, stored explicit choice).
- Legacy URLs: all historical biosked.fr and inherited biosked.com slugs 301 via
  `astro.config.mjs` redirects (Astro emits meta-refresh pages on GitHub Pages;
  `public/_redirects` is generated by `node scripts/generate-redirects.mjs` for a
  future host with native redirect support).
- HubSpot: demo form FR = legacy French form `a28f608c…`; other locales share the
  EN forms until per-locale clones exist (labels in HubSpot). Thank-you pages per
  locale; GA4 events `demo_form_submit` / `quote_form_submit` / `whitepaper_unlock`.
